Understanding HTML Structure

HTML is like the skeleton of a webpage - it provides structure and meaning

What is HTML?

HTML stands for HyperText Markup Language
HyperText: Text that can link to other text (like clicking links on websites)
Markup: A way to "mark up" text to give it meaning and structure
Language: A set of rules and syntax that browsers understand

üè† Think of HTML Like Building a House

üèóÔ∏è Building a House

  • Foundation: DOCTYPE and html tag
  • Frame: head and body sections
  • Rooms: Different content sections
  • Labels: Signs that say "Kitchen", "Bedroom"
  • Contents: Furniture and decorations
‚Üí

üåê Building a Webpage

  • Foundation: DOCTYPE declaration
  • Frame: html, head, body tags
  • Rooms: header, main, aside, footer
  • Labels: Headings (h1, h2, h3)
  • Contents: Text, images, links

Essential HTML Document Structure

Basic HTML Structure 
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Page Title</title>
</head>
<body>
    <!-- Your content goes here -->
</body>
</html>

Breaking Down Each Part:

<!DOCTYPE html>
Document Type Declaration

Tells the browser this is an HTML5 document

<html lang="en">
Root Element

Contains all content. "lang" specifies the language

<head>
Document Head

Contains metadata - info about the page, not visible content

<meta charset="UTF-8">
Character Encoding

Ensures text displays correctly (supports emojis, special characters)

<title>
Page Title

Shows in browser tab, search results, bookmarks

<body>
Document Body

Contains all visible content that users see

üîë Key Concepts to Remember

üìñ

Tags and Elements

Tags are the markup (like <h1>). Elements include the opening tag, content, and closing tag.

<h1>This is a heading</h1>
üîó

Nesting

Elements can contain other elements. Always close tags in the reverse order you opened them.

<p><strong>Bold text</strong></p>

Attributes

Provide additional information about elements. Written inside the opening tag.

Semantic HTML: Meaning Matters

Using the right tags for the right content makes websites better for everyone

What is Semantic HTML?

Semantic HTML means using HTML tags that have meaning, not just appearance. Instead of just making text look like a heading, we use heading tags that actually mean it's a heading.

‚ùå Non-Semantic vs ‚úÖ Semantic

‚ùå Not Semantic (Bad)

<div style="font-size:24px; font-weight:bold;">
    Article Title
</div>
<div>
    This is the article content...
</div>
<div style="font-size:18px; font-weight:bold;">
    Section Title
</div>

Problems:

  • Screen readers can't understand structure
  • Search engines don't know what's important
  • Hard to maintain and style

‚úÖ Semantic (Good)

<article>
    <h1>Article Title</h1>
    <p>This is the article content...</p>
    
    <section>
        <h2>Section Title</h2>
        <p>Section content...</p>
    </section>
</article>

Benefits:

  • Clear structure for screen readers
  • Better SEO (search rankings)
  • Easier to style and maintain

Important Semantic Tags

<header>
Top section of page/section (logo, navigation)
Use for: Site header, article header
<nav>
Navigation links
Use for: Menus, breadcrumbs
<main>
Main content of the page
Use for: Primary page content (only one per page)
<article>
Self-contained content
Use for: Blog posts, news articles
<section>
Thematic grouping of content
Use for: Chapters, tabs
<aside>
Sidebar or related content
Use for: Sidebars, callout boxes
<footer>
Bottom section (copyright, contact)
Use for: Site footer, article footer
